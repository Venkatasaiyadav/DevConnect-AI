// Rust Cosine Distance Vector Logic (Fixed) pub fn cosine_similarity(a: &[f32], b: &[f32]) -> Option<f32> { if a.len() != b.len() || a.is_empty() { return None; } let dot: f32 = a.iter().zip(b.iter()).map(|(x, y)| x * y).sum(); let norm_a: f32 = a.iter().map(|x| x * x).sum::<f32>().sqrt(); let norm_b: f32 = b.iter().map(|x| x * x).sum::<f32>().sqrt(); if norm_a == 0.0 || norm_b == 0.0 { return None; } Some(dot / (norm_a * norm_b)) }

React 19 Server Components re-rendering infinitely on query changes?

Hey everyone, I'm hitting a strange behavior in Next.js App Router (React 19 beta). When I call `router.push('/feed?filter=latest')` from a Client Component inside a Server Component wrapper, it re-fetches the entire page layout twice, leading to state loss. Below is the simplified structure of my page:

tsx 
// app/feed/page.tsx (Server Component)
export default async function FeedPage({ searchParams }) {
  const data = await fetchData(searchParams.query);

  return (
    <div>
      <FilterTabs /> {/* Client component with router.push */}
      <FeedList items={data.items} />
    </div>
  );
}

Has anyone encountered this state resetting? What's the best way to handle localized page parameter updates without retriggering root Suspense?
